# 要件書: ワリカンさん (Split Bill Calculator on Nostr)

## 1. プロジェクト概要

**ワリカンさん** は、Nostrプロトコルを活用した分散型の割り勘計算Webアプリケーション。
サーバーレス・アカウント不要で複数人がリアルタイムに割り勘精算を共有・同期できる。

---

## 2. 機能要件

### 2.1 コア機能

| ID    | 機能                                            | 優先度 |
| ----- | ----------------------------------------------- | ------ |
| FR-01 | メンバー管理（追加・表示）                      | 必須   |
| FR-02 | 支出登録（金額・支払者・説明）                  | 必須   |
| FR-03 | 割り勘計算（最小決済数アルゴリズム）            | 必須   |
| FR-04 | 精算リスト表示（誰が誰にいくら払うか）          | 必須   |
| FR-05 | スタンドアローンモード（ローカルのみ動作）      | 必須   |
| FR-06 | 同期モード（Nostrリレー経由でリアルタイム共有） | 必須   |
| FR-07 | 招待リンク生成と共有                            | 必須   |
| FR-08 | 精算のロック（確定・ファイナライズ）            | 必須   |
| FR-09 | 複数通貨対応（JPY / USD）                       | 必須   |
| FR-10 | ビットコイン価格取得・Sats換算表示              | 推奨   |
| FR-11 | Lightning Network決済（LNURL-pay）              | 推奨   |

### 2.2 スタンドアローンモード（FR-05）

- アカウント・Nostr接続不要
- SessionStorageにメンバー・支出データを永続化
- ページリロード後もデータ保持（セッション中のみ）
- オフラインでも動作可能

### 2.3 同期モード（FR-06）

- オーナーが精算セッションを作成しリレーに発行
- 招待リンク（URLパラメータ）で参加者を招待
- 参加者はリンクをブラウザで開くだけで参加可能（アカウント不要・エフェメラル鍵ペア）
- 全参加者がリアルタイムで同じデータを閲覧
- オーナーのみメンバー追加・精算ロックが可能
- 招待トークン所持者は支出を追加可能

### 2.4 招待・アクセス制御（FR-07）

- オーナーが招待URL（`?s={settlementId}&t={inviteToken}`）を生成
- inviteToken は32バイトのランダムな hex 文字列
- inviteToken のハッシュ（`SHA256(inviteToken)`）のみリレーに公開
- 招待リンクを持つ人は誰でも参加可能

### 2.5 精算ロック（FR-08）

- オーナーのみロック操作が可能
- ロック時に「採用する支出イベントID」を指定して確定
- ロック後は採用済みイベント以外の支出は無効とみなす
- ロック後は新規支出の追加不可

### 2.6 通貨・決済（FR-09 〜 FR-11）

- JPY / USD の切り替え表示
- CoinGecko APIからビットコイン価格をリアルタイム取得
- Sats（Satoshi）換算額の表示
- 受取人が Lightning Address（lud16）を Nostrプロフィールに設定していれば LNURL-pay 決済が可能
- 決済用QRコードまたは決済リンクの表示

## 3. セキュリティ要件

| ID    | 要件                                                                        |
| ----- | --------------------------------------------------------------------------- |
| SR-01 | 招待トークン（inviteToken）はリレーへ送信しない（ハッシュのみ公開）         |
| SR-02 | Capability（CAP）による支出追加の認可: `CAP = SHA256(inviteToken + pubkey)` |
| SR-03 | エフェメラル鍵ペアによる匿名参加（個人情報の登録不要）                      |
| SR-04 | オーナーのみ精算ロック・メンバー管理が可能                                  |
| SR-05 | Capabilityは行為者ごとに異なる値となり、リプレイ攻撃を防止                  |

---

## 4. 非機能要件

| ID     | 要件                                                              |
| ------ | ----------------------------------------------------------------- |
| NFR-01 | サーバーレス（Nostrリレーのみ使用。専用バックエンドサーバー不要） |
| NFR-02 | プライバシー（ユーザーアカウント・個人情報登録不要）              |
| NFR-03 | レスポンシブUI（モバイルファースト設計）                          |
| NFR-04 | アクセシビリティ（Radix UI使用・WCAG 2.1準拠）                    |
| NFR-05 | 型安全性（TypeScript strict mode）                                |
| NFR-06 | コード品質（Biomeによるlint/format・Vitestによるユニットテスト）  |
| NFR-07 | CoinGecko APIのレート制限対応（429応答時は1分間クールダウン）     |

---

## 5. ユーザーロール

| ロール   | 説明                             | できること                                         |
| -------- | -------------------------------- | -------------------------------------------------- |
| オーナー | 精算セッションを作成したユーザー | セッション作成・メンバー追加・精算ロック・支出追加 |
| 参加者   | 招待リンクから参加したユーザー   | 支出追加・精算閲覧                                 |
| 閲覧者   | セッションIDのみ知るユーザー     | 精算閲覧のみ                                       |
